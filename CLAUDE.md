## Project Purpose

This is a **QA agent benchmark**. TalentFlow is an HR management SPA with **100 intentionally planted bugs** across its frontend. The `auto_research/` pipeline trains specialized QA agents to find those bugs, evaluates their performance against ground truth, and iteratively optimizes agent prompts.

## Auto-Research Pipeline

### Setup
```bash
pip install -e .[browser-use]

# DeepInfra (default provider):
export DEEPINFRA_API_KEY="your-key-here"   # https://deepinfra.com/dash/api_keys

# OR use OpenAI Codex (legacy):
export AUTO_RESEARCH_LLM_PROVIDER=codex
python -c "from browser_use.utils.cli import openai_oauth2_login; openai_oauth2_login()"
```

### Running
```bash
# Full run (default: browser-use mode, 5 iterations)
python -m auto_research.main --iterations 5

# Single agent
python -m auto_research.main --single edge_case --iterations 3

# Different modes: source (code-only), browser (playwright), browser-use (autonomous)
python -m auto_research.main --mode source --iterations 1
python -m auto_research.main --mode browser --iterations 1

# Export trained prompts as Codex skills
python -m auto_research.main --iterations 5 --export

# Custom app URL (if already running)
python -m auto_research.main --app-url http://127.0.0.1:3013 --iterations 1
```

## Architecture

### Frontend (the benchmark app)

Vanilla JS single-page app with hash-based routing. No frameworks, no build step.

- `index.html` — Shell: sidebar, header, modals, footer
- `style.css` — All styles (~3000 lines)
- `js/app.js` — Router, event handlers, form submission logic
- `js/store.js` — In-memory state management + CRUD operations + validation
- `js/render.js` — HTML generation/templating for all pages
- `js/data.js` — Seed data with `BUG-XXX` comments marking intentional bugs

### Bug catalog

`BENCHMARK.md` contains all 100 bugs with IDs, severity, category, location, trigger, and expected agents. `auto_research/ground_truth/bugs.json` is the parsed machine-readable version (generated by `ground_truth/parser.py`).

**Bug categories**: Form Validation (15), Functional (20), UI/Visual (15), Accessibility (15), Responsive (13), Security (10), Interaction (12).

### Auto-Research Pipeline (`auto_research/`)

**Training loop** (`main.py`): Run agents → evaluate vs ground truth → analyze gaps → optimize prompts → repeat.

**6 specialist agents** (`agents/prompts/`): happy_path, edge_case, accessibility, error_recovery, viewport, security. Plus a reviewer agent for consolidation.

**3 execution modes** (`runner/harness.py`):
- `source` — Agents analyze concatenated source code directly (~10s/agent)
- `browser` — Playwright crawl captures screenshots/state, then agents analyze (~2 min total)
- `browser-use` — Agents autonomously control Chrome via CDP (~5-10 min, default)

**Evaluation** (`runner/evaluator.py`, `ground_truth/matching.py`): LLM-as-judge matches findings to ground truth bugs. Calculates precision/recall/F1 per agent and overall.

**Prompt optimization** (`optimizer/`): Gap analyzer classifies missed bugs (blind_spot, wrong_file, shallow_analysis, pattern_miss). Prompt generator uses a meta-prompt to improve agent prompts for the next iteration.

**Export** (`exports/export.py`): Trained prompts exported as reusable Codex skills.

### Key config (`config.py`)

**LLM Provider** (`AUTO_RESEARCH_LLM_PROVIDER`): `deepinfra` (default) or `codex`. Controls which backend is used for all LLM calls.

| Variable | DeepInfra default | Codex default | Purpose |
|---|---|---|---|
| `DEEPINFRA_API_KEY` | *(required)* | — | DeepInfra API key |
| `DEEPINFRA_BASE_URL` | `https://api.deepinfra.com/v1/openai` | — | DeepInfra endpoint |
| `AUTO_RESEARCH_ITERATION_MODEL` | `meta-llama/Llama-3.3-70B-Instruct` | `gpt-5.4` | Agent iteration model |
| `AUTO_RESEARCH_VALIDATION_MODEL` | `meta-llama/Llama-3.3-70B-Instruct` | `gpt-5.4` | Judge/matching model |
| `AUTO_RESEARCH_OPTIMIZER_MODEL` | `meta-llama/Llama-3.3-70B-Instruct` | `gpt-5.4` | Prompt optimization model |
| `BROWSER_USE_LLM_PROVIDER` | `deepinfra` | `openai` | Browser-use agent provider |
| `BROWSER_USE_LLM_MODEL` | `meta-llama/Llama-3.3-70B-Instruct` | `gpt-5.4-mini` | Browser-use agent model |

Other environment variables: `AUTO_RESEARCH_RUN_MODE`, `AUTO_RESEARCH_APP_URL`, `BROWSER_USE_CDP_URL`.

Training defaults: MAX_ITERATIONS=5, MIN_F1_IMPROVEMENT=0.01, NO_IMPROVEMENT_PATIENCE=3.

### Data schemas (`agents/schemas.py`)

Pydantic models: Finding, Bug, AgentReport, EvalResult, RunResult. Findings have: id, severity, category, title, description, file, function_or_section, line_evidence, impact, reproduction.
